Usage Guide

Everything you need to configure, run, and integrate sensei.

CLI Commands

The sensei binary handles setup, daemon lifecycle, and indexing.

🔧

Setup

sensei configure Detect installed AI coding platforms and generate config
sensei install Install hooks, skills, and MCP server for all platforms
sensei install --acp claude-code Install for a specific platform only
sensei uninstall Remove sensei from all configured platforms
🔍

Indexing

sensei scan ~/Developer Scan a folder and index all repos found
sensei add-lib react Add a library's docs (auto-discovers llms.txt)
sensei add-lib kavach --url https://... Add a library with explicit doc URL

Daemon Management

The senseid daemon indexes repos and serves the local API.

senseid Start in foreground (logs to stdout)
senseid start Start as background daemon
senseid stop Stop the background daemon
senseid status Show version and PID
senseid logs Tail the last 50 lines
senseid clear-logs Clear the log file
🍺

Homebrew service

Use brew services start sensei to run senseid as a login service.

Architecture

senseid

Indexer daemon — watches repos, builds code graph, serves HTTP API

sensei-mcp

MCP adapter — translates AI platform tool calls into HTTP requests

sensei CLI

Manages installation, config, scanning — talks to senseid over HTTP

Sensei Desktop

Tauri app — project navigator, graph viewer, indexer control

🔒

Local-first, private by default

All data is stored at ~/.sensei/. Nothing is sent to external servers. The daemon stores its PID, logs, SQLite database, and per-repo graph data locally.
