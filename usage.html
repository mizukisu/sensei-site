Usage Guide

CLI commands, daemon management, and MCP tools.

CLI Commands

The sensei CLI manages installation, configuration, and scanning.

sensei configure Detect installed AI coding platforms and generate config
sensei install Install sensei for all configured platforms (hooks, skills, MCP)
sensei install --acp claude-code Install for a specific platform only
sensei uninstall Remove sensei from all configured platforms
sensei start Start the senseid daemon (default port 7744)
sensei start --port 8800 Start on a custom port
sensei stop Stop the running daemon
sensei status Show daemon status (version, PID, uptime)
sensei scan ~/Developer Scan a folder and index all repos found
sensei add-lib react Add a library's docs (auto-discovers llms.txt)
sensei add-lib kavach --url https://... Add a library with explicit doc URL

Daemon Management

The senseid daemon runs the indexer and serves the API on port 7744.

senseid Start daemon in foreground
senseid start Start daemon in background
senseid stop Stop the background daemon
senseid status Show daemon version and PID
senseid logs Tail the last 50 lines of the daemon log
senseid clear-logs Clear the log file
Homebrew service: If installed via Homebrew, you can also use brew services start sensei to run senseid as a background service that starts on login.

MCP Tools

When connected via sensei-mcp, your AI coding platform gets these tools:

get_session_context Resume from last checkpoint, load repo orientation
context_pack Load only the symbols relevant to a query (saves tokens)
search Semantic search across the entire codebase
load_context Load a specific file with symbol extraction
take_snapshot Record progress for interruption recovery
checkpoint Close a session with outcome tracking (FTR scoring)

Architecture

┌─────────────┐     MCP      ┌────────────┐     HTTP     ┌──────────┐
│ Claude Code │ ──────────── │ sensei-mcp │ ──────────── │ senseid  │
│ Cursor      │  (stdio)     │            │  (:7744)     │ (daemon) │
│ Windsurf    │              └────────────┘              └──────────┘
└─────────────┘                                               │
                                                         ┌────┴─────┐
                                                         │ SQLite   │
                                                         │ Graph DB │
                                                         └──────────┘

senseid — The indexer daemon. Watches your repos, builds a code graph in SQLite, and serves an HTTP API.

sensei-mcp — A thin MCP adapter. Translates MCP tool calls from your AI platform into senseid HTTP requests.

sensei — The CLI. Manages installation, configuration, and scanning. Talks to senseid over HTTP.

Data storage

All data is stored locally at ~/.sensei/. Nothing is sent to external servers. The daemon stores its PID, logs, SQLite database, and per-repo graph data in this directory.
